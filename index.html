    Rules for XML

    In Document > CstmrCdtTrfInitn

    There is two types of XML Schema extension:

    1. GrpHdr (Group Header – main information about payment(-s), initiator name)

    2. PmtInf (Payment Information - debtor and receiver)

    Required Parameters

    Category Parameter(-s) Description Explanation Naming in XLS
    CstmrCdtTrfInitn > GrpHdr Main information Group Header
    MsgId must be generated unique (from 7-35 symbols) Autogenerated
    CreDtTm datetime of generation date (format example: 2019-03-27T13:15) Autogenerated
    NbOfTxs number of transactions (number of payments from XLS) Count number of payments list
    CtrlSum control sum is amount of all payments (total) Count total amount of all payments
    InitgPty it is initiator information block
    InitgPty > Nm Initiator name, can be organisation or person example: [Name Surname / Company Name Ltd.] Initiator name
    CstmrCdtTrfInitn > PmtInf Payment information this could be multiple (each block)
    PmtInfId must be generated unique (from 7-35 symbols) Autogenerated
    PmtMtd must be TRF TRF
    NbOfTxs must be 1 1
    CtrlSum there must be the amount of exact payment from input Amount Amount
    PmtTpInf > SvcLvl > Cd must be SEPA SEPA
    ReqdExctnDt date when to do payment from input Initiation date Initiation date
    Dbtr Debtor name from input Name (debtor) Name (debtor)

    Optional Parameters

    Category Parameter(-s) Description Explanation Naming in XLS
    CstmrCdtTrfInitn > PmtInf Payment information this could be multiple (each block)
    Dbtr > PstlAdr > Ctry Debtor country from input Country (debtor) it must go with Address (debtor) Country (debtor)
    Dbtr > PstlAdr > AdrLine Debtor address line from input Address (debtor) it must go with Country (debtor) Address (debtor)
    Dbtr > Id > [PrvtId / OrgId] > Othr > Id Debtor code from input Code (debtor) it must go with Type (debtor) and Identification type (debtor) Code (debtor)
    Dbtr > Id > [PrvtId / OrgId] > Othr > SchemeNm > Cd Debtor identification type from input Identification type (debtor) it must go with Type (debtor) and Code (debtor) Identification type (debtor)
    DbtrAgt > FinInstnId > BIC Debtor bank code from input BIC (debtor) BIC (debtor)
    CdtTrfTxInf > Cdtr > PstlAdr > Ctry Debtor country from input Country (receiver) it must go with Address (receiver) Country (receiver)
    CdtTrfTxInf > Cdtr > PstlAdr > AdrLine Debtor address line from input Address (receiver) it must go with Country (receiver) Address (receiver)
    CdtTrfTxInf > Cdtr > Id > [PrvtId / OrgId] > Othr > Id Debtor code from input Code (receiver) it must go with Type (receiver) and Identification type (receiver) Code (receiver)
    CdtTrfTxInf > Cdtr > Id > [PrvtId / OrgId] > Othr > SchemeNm > Cd Debtor identification type from input Identification type (receiver) it must go with Type (receiver) and Code (receiver) Identification type (receiver)
    CdtTrfTxInf > CdtrAgt > FinInstnId > BIC Receiver bank code from input BIC (receiver) BIC (receiver)
