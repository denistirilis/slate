    Introduction

    Welcome to the IBS KYC API! You can use our API to access IBS Simplified Forms API endpoints, which can get information on user account details and forms queries in our database.

    We have language bindings in PHP, Shell and JavaScript! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
  -H "Authorization: Bearer authenticationToken"

    $curl = curl_init("api_endpoint_here");

curl_setopt(
  $curl, 
  CURLOPT_HTTPHEADER, 
  array(
    "Content-Type: application/json" , 
    "Authorization: Bearer authenticationToken"
    ))

    const authToken = require('api_endpoint_here');

let api = authToken.authorize('authenticationToken');

    Make sure to replace authenticationToken with your API key.

    Bearer Authentication key allow access to the API. You can get authentication token by contacting our support.

    Authentication token must be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer authenticationToken

    You must replace authenticationToken with your personal API key.

    This endpoint creates form query for user.

    HTTP Request

    POST https://ibsettle.com/kyc/api/forms/create

    Required POST Parameters

    Parameter Description Requirements
    token Token for user detect hash token code which you get after User creation
    name User Name and Surname minlength => 1, maxlength => 150
    email User Email Address E-mail validation
    phone User Phone Number Phone validation
    residenceAddress User Residence Address maxlength => 150
    residencePostalCode User Residence Postal Code maxlength => 20
    residenceState User Residence State maxlength => 150
    residenceCity User Residence City maxlength => 100
    residenceCountry User Residence Country Remitter country code (2-letter ISO 3166-2 country code)
    personBirthDate User Birth of Date Format: 1990-01-01
    personCitizenship User Citizenship Remitter country code (2-letter ISO 3166-2 country code)

    Optional POST Parameters

    Parameter Description
    personCode User person code
    personPE User Politicaly Exposed (Yes / No)
    additionalDetailsPE If User Politicaly Exposed (Yes) then fill additional details
    sourceOfFunds User source of funds
    merchantName Merchant Name

    Errors

    API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The request is hidden for administrators only.
    404 Not Found -- The specified request could not be found.
    405 Method Not Allowed -- You tried to access invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    429 Too Many Requests -- You're requesting too many requests!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
